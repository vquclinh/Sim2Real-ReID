## 1. Project Overview

**Task**: Text-Based Person Anomaly Search. Given a natural-language query describing a person's behavior (normal or anomalous), rank the top-10 images from a 36,773-image real-world gallery.

**Challenge**: AIC 2026 Track 4 (ECCV 2026 Workshop). Evaluation metric is **mAP** (mean Average Precision), secondary R@1/R@5/R@10. Single-positive setting: each query has exactly 1 ground-truth image.

**Dataset — PAB (Pedestrian Anomaly Behavior)**:

| Split | Count | Note |
|-------|-------|------|
| Train | 1,013,606 (image, caption) pairs | Synthetic (diffusion-generated) — Sim2Real gap |
| Gallery (test) | 36,773 images | Real-world |
| Query (test) | 1,978 captions | Real-world descriptions |

**Core difficulty**: Train on synthetic images, infer on real images → domain shift.

**Foundation paper**: "Hybrid, Unified and Iterative: A Novel Framework for Text-based Person Anomaly Retrieval" (Tien-Huy Nguyen et al., ACM WWW 2025) — achieved R@1=89.23% on PAB.

## 4. Current Pipeline (What Was Actually Submitted)

### 4.1 Submitted Pipeline — Zero-Shot PE-G14

```
query_index.txt (1978 query IDs, defines submission ORDER)
query_text.json (JSONL: query_index → caption)
        │
        ▼
    Align captions to query_index.txt order
        │
        ▼
    PE-Core-G14-448 (Meta, 1.8B params, pretrained)
    ┌───────────────────────────────────┐
    │  Text encoder: encode 1978 queries │  → (1978, 1280) fp16 L2-norm
    │  Image encoder: encode 36,773 gallery│  → (36773, 1280) fp16 L2-norm
    └───────────────────────────────────┘
        │
        ▼
    Cosine similarity (chunked, 256 queries/chunk)
    sim = Q @ G.T   → (1978, 36773)
        │
        ▼
    torch.topk(sim, k=10, dim=1) → top10_indices
        │
        ▼
    gallery_ids[top10_indices]  ← filename stems (no extension)
        │
        ▼
    answer.txt: 1978 lines × 10 gallery IDs space-separated
    answer.zip
```

**Type**: Zero-shot, single model, cosine similarity only. No training, no ensemble, no re-ranking.

### 4.2 Designed-but-not-run Pipeline — Full Upgrade Pipeline

The `notebooks_upgrade/` directory contains a complete 10-notebook pipeline (described in `ARCHITECTURE.md`) that has been designed but not yet executed. When run, it would implement:

```
00 Manifest QC → canonical parquets + val splits
        │
        ├──── 01a PE-G14 features (gallery+queries+val)
        │
        ├──── 01b ViTPose++ keypoints [OPTIONAL]
        │
        ├──── 02 UIT train (Swin-B + BERT, 22 epoch, 4 losses)
        │
        └──── 03 LHP-PE-G14 LoRA train (freeze 99.5%, LoRA r=16, 3 epoch)
                    │
    04 UIT Inference (LHP-guided Algorithm 1 + ITM rerank)
    05 BLIP-2 ITC + ITM rerank (top-1024)
    06 CLIP ViT-L/14@336 cosine
    07 PE-G14 score matrix (from 01a)
        │
    08 k-reciprocal Jaccard rerank (k1=20, k2=6, λ=0.3) per model
        │
    09 Adaptive 4-way ensemble (per-query z-norm + softmax weights)
                    │
                answer.zip
```

## 11. Known Risks / Possible Bugs

### ✅ Confirmed Safe (zero_shot notebook)

1. **answer.txt order follows query_index.txt**: `submission_qids = [ln.strip() for ln in QUERY_INDEX_TXT.read_text().splitlines() if ln.strip()]` — correct, uses exact file order.

2. **Image names without extensions**: `gallery_ids = [p.stem for p in gallery_paths]` — stems only, no extension. Format matches `Document/answer.txt`.

3. **Gallery ordering deterministic**: `gallery_paths = sorted(GALLERY_DIR / n for n in names)` — lexicographic sort over shell `ls` output. Deterministic as long as Drive layout doesn't change.

4. **Features normalized before similarity**: Both gallery and query embeddings go through `F.normalize(..., dim=-1)` before cosine computation.

5. **Similarity computed correctly**: `Q @ G.T` with L2-normalized vectors = cosine similarity. Sanity check vs Milvus L2 confirms equivalence.

6. **Top-10 sorted correctly**: `torch.topk(sims, k=10, dim=1)` returns descending order by default — correct.

7. **No data leakage**: Gallery images from `name-masked_test-set/gallery/`, no overlap with training data.

### ⚠️ Potential Risks (upgrade pipeline — not yet run)

8. **CRITICAL — `proj_head` bug already fixed**: ARCHITECTURE.md §9.1 documents that a previous version of `03_lhp_peg14_train.ipynb` trained with `proj_head` but ran inference without it. The current notebook version should have this fixed — **verify before running**.

9. **Algorithm 1 requires aligned IDs**: `04_uit_inference.ipynb` loads `scores_lhp.pt` and does `align_ids(lhp_payload, q_ids, g_ids)`. If the gallery ordering in 03 (LHP scores) differs from 04, scores will be misaligned. Both notebooks use the same `gallery_manifest.parquet` — verify they sort gallery IDs identically.

10. **Gallery-gallery embedding for k-reciprocal**: `08_kreciprocal_rerank.ipynb` needs gallery embeddings per model. Confirm that each model's gallery embeddings are saved and loaded — the notebook documentation says it uses `G_m` (gallery feature matrix) from the respective model's NPZ files.

11. **BLIP-2 scale mismatch**: BLIP-2 ITM outputs logits (cross-entropy scale) while CLIP/PE-G14 output cosine similarities [−1, 1]. The per-query z-normalization in `09` is critical. If scores are fused without z-norm, BLIP-2 dominates spuriously.

12. **k-reciprocal memory**: TOPK_RR=200, k1=20 — building full gallery-gallery GG matrix (36773, 36773) in FP32 requires ~5.4GB. Confirm it fits on target GPU.

13. **UIT cross-encoder memory**: `04` caches `(36773, ~50, 1024)` fp16 for all gallery token-level embeds ≈ 3.7GB CPU RAM. Fine on 80GB A100 (high-RAM mode), may OOM on standard 12GB CPU.
